# Demo Script

## Goal

Prepare a repeatable final demonstration for the project defense.

## Draft Flow

1. Present the project goal and architecture.
2. Show the repository structure and documentation.
3. Show local or staging application access.
4. Show HTTPS access through Traefik.
5. Show database persistence.
6. Show CI/CD pipeline execution.
7. Show infrastructure or deployment configuration.
8. Show logs, metrics, and dashboards.
9. Show backup and restore evidence.
10. Explain limitations and future improvements.

## Evidence To Prepare

- Working application URL.
- Pipeline run.
- Deployment logs.
- Monitoring dashboard.
- Backup and restore proof.
- Architecture diagram.
